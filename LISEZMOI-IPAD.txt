🐷 LA COCHETTE DORÉE — VERSION STANDALONE iPAD PRO (PWA)
=========================================================
Ton application Flask tourne INTÉGRALEMENT dans Safari grâce à
Pyodide (Python compilé en WebAssembly). Aucune modification de la
logique métier : app.py est exécuté tel quel, les 102 routes API
fonctionnent, la base SQLite vit dans l'iPad (IndexedDB).

INSTALLATION (une seule fois, avec internet)
--------------------------------------------
1. Héberge ce dossier COMPLET sur n'importe quel hébergement statique
   gratuit avec HTTPS. Le plus simple :
   • GitHub Pages : crée un dépôt, glisse ces fichiers, active Pages.
   • Ou Netlify (https://app.netlify.com/drop) : glisse le dossier → URL immédiate.
2. Sur l'iPad, ouvre l'URL dans SAFARI (pas Chrome).
3. Attends la fin du chargement (20–60 s la première fois).
4. Bouton Partager → « Sur l'écran d'accueil ».
5. Lance l'app depuis l'icône 🐷 : plein écran, HORS-LIGNE ensuite.

DONNÉES
-------
• La base démarre avec la copie de cochette.db incluse, puis vit
  dans l'iPad : chaque ajout/modif est sauvegardé automatiquement.
• Bouton flottant 💾 (bas droite) :
  ⬇︎ Exporter la base  → fichier cochette-AAAA-MM-JJ.db (sauvegarde !)
  ⬆︎ Restaurer une base → recharge un fichier .db exporté (écrase tout).
• IMPORTANT : exporte régulièrement. Si tu supprimes l'app ou vides
  les données Safari, la base locale est perdue.
• Pour synchroniser avec la version Windows/PC : exporte le .db ici
  et remplace le fichier cochette.db côté PC (ou l'inverse, via
  « Restaurer une base »).

LIMITES CONNUES
---------------
• Webhooks WhatsApp/Meta et page /commander : inertes en mode
  hors-ligne (aucun serveur public pour les recevoir) — normal.
• Premier démarrage : téléchargement ~25 Mo (mise en cache ensuite).
• Un seul appareil = une seule base ; pas de synchronisation auto.

MISE À JOUR DE L'APP
--------------------
Remplace les fichiers sur l'hébergement, puis dans sw.js incrémente
VERSION ('cochette-v1' → 'cochette-v2'). Recharge l'app une fois en ligne.

INSTALLATION 100 % HORS-LIGNE (sans hébergement web)
----------------------------------------------------
iPadOS ne peut pas exécuter de programme directement : il faut ouvrir
le dossier UNE première fois via un mini-serveur local sur l'iPad.
1. Installe (une fois, gratuit) l'app « WorldWideWeb » (The Iconfactory)
   depuis l'App Store — c'est la seule étape nécessitant une connexion.
2. Copie le dossier cochette-ipad sur l'iPad (câble USB, AirDrop,
   ou app Fichiers) — aucun internet requis.
3. Ouvre WorldWideWeb → choisis le dossier cochette-ipad → « Start Server ».
4. Ouvre l'adresse localhost affichée dans SAFARI.
5. Attends la fin du chargement, puis Partager → « Sur l'écran d'accueil ».
6. Tu peux fermer WorldWideWeb : l'icône 🐷 fonctionne ensuite seule,
   totalement hors-ligne (tout est en cache sur l'iPad).
Alternative équivalente : app « Koder » ou tout autre serveur local.
